# Monster Sprite Art Direction Skill

## Purpose

This skill defines the shared visual language for every monster in the game.

Its purpose is to ensure that radically different creatures still appear to belong to the same:

* game
* fictional universe
* hardware era
* art pipeline
* production team
* battle interface

This skill controls:

* pixel-art construction
* silhouette readability
* palette discipline
* outline behavior
* lighting and shading
* sprite staging
* perspective
* cross-view consistency
* technical export standards
* visual quality assurance
* originality
* creative design discipline
* developmental-form continuity
* Mega-Morphosis transformation direction

This skill does not prescribe:

* any particular body plan
* a standard limb count
* bilateral symmetry
* a head-and-torso structure
* a face
* eyes
* a mouth
* wings
* tails
* horns
* ears
* shells
* fur
* scales
* elemental markings
* accessories
* posture
* locomotion
* personality
* color scheme

No anatomical feature is assumed to exist unless it is chosen for the current creature.

# Core Art Direction

Create entirely original monsters in a polished late-2000s handheld monster-catching RPG pixel-art aesthetic.

The artwork should evoke the visual discipline of Nintendo DS-era creature-battling games without copying, tracing, recreating, or closely imitating any specific copyrighted creature, sprite, pose, or recognizable design.

The shared aesthetic should feel:

* colorful
* compact
* readable
* expressive
* game-ready
* deliberately hand-pixelled
* anime-influenced where appropriate
* visually cohesive
* mechanically clean
* designed for a limited-resolution battle interface

The creature must look designed directly as pixel art.

It must not look like a high-resolution illustration that was automatically reduced, pixelated, or filtered.

# Foundational Creative-Liberty Rule

The agent has broad creative authority over the creature's physical design unless the current brief establishes specific requirements.

The agent may decide:

* whether the creature has limbs
* how many limbs it has
* whether different limbs serve different functions
* whether it has a conventional head
* whether it has a recognizable face
* whether it has eyes
* how many eyes it has
* whether it has a mouth
* whether it is symmetrical
* how it moves
* whether it flies, swims, crawls, rolls, floats, burrows, climbs, walks, or remains rooted
* whether it has an internal, external, segmented, armored, fluid, botanical, mineral, spectral, mechanical, or hybrid structure
* whether it uses biological or non-biological anatomy
* whether it possesses any optional appendage or surface feature
* how its body expresses temperament, habitat, behavior, and elemental identity

Creative liberty must be used to create meaningful variation, not random complexity.

Every design decision should support at least one of the following:

* silhouette
* function
* habitat
* movement
* personality
* elemental identity
* defensive strategy
* offensive strategy
* ecological role
* developmental relationship
* temporary transformation logic
* visual storytelling

Do not assume that a monster should be:

* mammalian
* reptilian
* avian
* humanoid
* quadrupedal
* bipedal
* dragon-like
* pet-like
* conventionally cute
* conventionally fierce
* symmetrical
* vertebrate
* terrestrial

Do not automatically add familiar creature features merely because the brief leaves anatomy open.

In particular, do not default to adding:

* wings
* horns
* tails
* pointed ears
* paws
* claws
* fangs
* cheek markings
* crests
* spikes
* fur tufts
* glowing gems
* elemental symbols
* decorative appendages

Any such feature may be used when it strengthens the current design, but none should be treated as part of the global style.

The absence of a feature is an equally valid design decision.

A monster without wings, a tail, horns, limbs, visible eyes, or a conventional face can still fully belong to the project.

# Roster Diversity Safeguard

Before generating a new random creature, consider whether its major structural decisions are too similar to recently created monsters.

Compare:

* body plan
* limb count
* locomotion
* dominant silhouette
* head structure
* face placement
* appendage arrangement
* surface material
* posture
* size impression
* symmetry
* primary motif

When several recent creatures already share a trait, reduce the likelihood of using it again unless the brief specifically calls for it.

Examples:

* If recent creatures are mostly upright bipeds, consider a non-biped.
* If recent creatures have prominent wings, consider a grounded or floating design without wings.
* If recent creatures have tails, consider a body plan where a tail is absent or irrelevant.
* If recent creatures use animal-like faces, consider an unconventional sensory structure.
* If recent creatures are symmetrical, consider controlled asymmetry.
* If recent creatures are soft-bodied, consider armor, mineral, plant, or constructed materials.

Do not change the established art style to create variety.

Vary the creature architecture while preserving the rendering language.

# Controlled Randomness

Creative liberty does not mean arbitrary feature accumulation.

Use controlled randomness:

1. Choose one foundational body concept.
2. Choose one movement or behavioral concept.
3. Choose one visual motif.
4. Choose one material language.
5. Add only the anatomical features required to express those choices.
6. Remove features that do not contribute.
7. Simplify the result for sprite readability.

Avoid designs that feel like lists of unrelated traits.

Do not solve originality by attaching more parts.

# Universal Style Pillars

## 1. Strong Silhouette

Every monster must be recognizable from its outer shape alone.

Prioritize:

* a clear primary mass
* readable secondary masses
* intentional negative space
* distinct movement or posture
* one dominant silhouette idea
* clean visual hierarchy

Avoid:

* tangled forms
* excessive protrusions
* decorative clutter
* unintentionally merged body regions
* outlines with no visual rhythm
* features too small to affect recognition

At thumbnail size, the creature should remain understandable.

A readable silhouette does not require a conventional head, limbs, or face.

---

## 2. Species-Appropriate Proportions

Use stylized proportions appropriate to the current creature.

Proportions may be:

* compact
* elongated
* squat
* top-heavy
* bottom-heavy
* radial
* segmented
* delicate
* massive
* unstable
* floating
* asymmetrical

Do not force every monster into a large-head, small-body template.

Expressiveness may come from:

* posture
* shape
* eye placement
* body angle
* appendage position
* movement
* surface deformation
* silhouette
* color contrast

The shared style comes from pixel construction and art direction, not identical proportions.

---

## 3. Sprite-First Design

Design for the final sprite resolution from the beginning.

Every major feature must survive at actual display size.

Before retaining a detail, ask:

> Does this detail communicate something important at native sprite scale?

Retain details that communicate:

* species identity
* anatomy
* behavior
* material
* habitat
* role
* movement
* personality
* developmental stage
* transformed-state identity
* battle function

Remove details that only create noise.

Avoid relying on:

* tiny ornaments
* miniature patterns
* individual strands
* complex engraving
* subtle gradients
* low-contrast differences
* details visible only when enlarged

---

## 4. Hand-Placed Pixel Construction

The artwork must appear deliberately constructed pixel by pixel.

Use:

* crisp pixel boundaries
* deliberate clusters
* controlled stair-stepping
* intentional corners
* clear material separation
* compact highlights
* clustered shadows
* purposeful outline variation

Avoid:

* automatic pixelation filters
* noisy dithering
* random isolated pixels
* inconsistent pixel density
* partially smoothed contours
* resampling artifacts
* blurry edges

Pixel clusters should describe form and material.

They should not imitate texture indiscriminately.

---

## 5. Limited Palette

Use a compact, vibrant palette designed for the current monster.

A typical creature may use:

* one outline family
* one dominant material ramp
* one or two supporting ramps
* limited accent colors
* a small sensory-feature palette when needed

Each major material should generally use a two-to-four-step ramp:

1. shadow
2. base
3. light
4. optional highlight

Not every material needs all four levels.

Avoid adding colors when an existing ramp can communicate the same form.

Colors must remain distinguishable at sprite scale.

There is no universal creature palette.

Do not repeat the same dominant colors across the roster without a design reason.

---

## 6. Compact Cel-Style Shading

Use simplified cel-style lighting translated into pixel clusters.

Shading should:

* clarify volume
* separate overlapping regions
* explain material
* reinforce the pose
* preserve sprite readability
* remain consistent between views

Prefer a consistent upper-left or upper-front-left light source unless the project specifies otherwise.

Do not use:

* soft airbrushing
* realistic global illumination
* smooth gradients
* painterly blending
* photographic lighting
* glossy 3D rendering
* excessive rim light
* bloom
* volumetric effects

---

## 7. Controlled Outlines

Use crisp, generally one-pixel outlines at native sprite scale.

Outlines should be:

* dark enough to separate forms
* consistent in visual weight
* clean around the silhouette
* selectively adapted to local colors
* readable against the transparency key

Pure black may be used sparingly.

Dark colored outlines are often preferable when they improve palette harmony.

Avoid:

* thick uniform cartoon borders
* inconsistent line weight
* anti-aliased pixels
* semi-transparent edges
* outlines too close to the background key
* unnecessary internal contour lines

---

## 8. Expressive Design Without Mandatory Faces

A monster should communicate temperament clearly, but it does not need a traditional face.

Expression may be conveyed through:

* eyes
* eyelids
* mouth shape
* posture
* body compression
* body expansion
* orientation
* limb position
* appendage movement
* glow intensity
* surface pattern
* silhouette tension
* asymmetry
* movement direction

When a conventional face exists, keep it readable and economical.

When no conventional face exists, establish another clear focal system.

Do not give every creature the same large-eyed expression.

# Battle Sprite Requirements

When a standard battle pair is requested, create:

1. one front-facing three-quarter battle sprite
2. one back-facing three-quarter battle sprite

These labels describe camera orientation, not required anatomy.

A creature without a conventional front or back should still be presented through two logically opposing battle viewpoints.

Both views must depict the exact same design.

## Front Battle View

The front battle sprite should:

* present the creature clearly
* communicate battle behavior
* show its primary visual idea
* preserve readable negative space
* avoid a flat diagrammatic presentation
* make orientation understandable

## Back Battle View

The back battle sprite should:

* reconstruct the same body from the opposing viewpoint
* preserve structural logic
* maintain the same proportions
* maintain palette and lighting
* reveal hidden surfaces consistently
* avoid inventing additional structures

The back view must not be a mirrored front view.

---

# Cross-View Consistency

The front and back sprites must agree on every feature that exists.

Check:

* body regions
* structural proportions
* number of appendages
* appendage placement
* segmentation
* surface pattern
* material
* openings
* sensory organs
* decorative structures
* armor or shell boundaries
* palette
* outline behavior
* lighting direction
* sprite scale
* apparent age or developmental stage

Do not run a checklist for features the creature does not possess.

For example:

* Do not expect a tail when none was designed.
* Do not expect eyes when the creature senses through another structure.
* Do not expect limbs when movement is fluid, rolling, floating, rooted, or serpentine.
* Do not expect bilateral symmetry when asymmetry is intentional.

Consistency means preserving the chosen anatomy, not forcing conventional anatomy.

---

# Structural Connection Discipline

Every body structure must connect or relate coherently.

Check:

* whether functional structures emerge from believable regions
* whether segmented pieces align
* whether armor plates overlap logically
* whether floating components are intentionally floating
* whether disconnected components have a clear magical, mechanical, or biological relationship
* whether the center of mass supports the pose
* whether the creature's movement method is visually plausible
* whether rear structures correspond with the front view

A floating structure is not automatically an error.

It becomes an error when the design gives no indication that floating separation is intentional.

# Elemental Design Restraint

Elemental identity should not automatically be represented by literal symbols.

An electric creature does not inherently need lightning marks.

A fire creature does not inherently need a flame-shaped tail.

A water creature does not inherently need fins.

A flying creature does not inherently need feathered wings.

An earth creature does not inherently need rocks attached to its body.

A plant creature does not inherently need leaves.

Elemental or thematic identity may instead be conveyed through:

* movement
* material
* palette
* anatomy
* behavior
* silhouette
* attack posture
* habitat adaptation
* surface response
* energy flow

Use literal motifs only when they strengthen the design.

# Creature Form Taxonomy

Use neutral, project-owned terminology when describing changes between creature forms.

## Permanent Developmental Stages

A permanent stage represents growth, maturation, metamorphosis, adaptation, or another lasting biological, magical, elemental, artificial, or ecological change.

Recommended language includes:

* earlier stage
* juvenile stage
* intermediate stage
* mature stage
* next developmental stage
* metamorphic stage
* specialized branch form
* final mature form

Do not require all species to follow the same number of stages or the same biological process.

A permanent developmental form should feel like a related member of the same creature line while possessing enough structural change to justify its own gameplay identity.

## Standard Forms and Variants

A standard form or variant may represent:

* environmental adaptation
* regional adaptation
* seasonal state
* sex-linked variation
* artificial modification
* elemental attunement
* habitat specialization
* alternate battle role
* story-specific condition

A variant is not automatically stronger than the base creature.

## Mega-Morphosis

**Mega-Morphosis** is the project's term for a temporary, high-output battle transformation of an existing creature.

A Mega-Morphosis is:

* temporary
* visibly more powerful
* mechanically specialized
* derived from one established base creature
* recognizable as that same creature
* distinct from ordinary developmental stages
* visually dramatic without abandoning sprite readability

A Mega-Morphosis is not:

* a permanent developmental stage
* a separate species
* an unrelated redesign
* a simple size increase
* a recolor
* generic armor placed over the creature
* random feature accumulation
* an excuse to abandon the base creature's visual logic

When prompting an image model, describe it as a **temporary empowered transformation called Mega-Morphosis**. Do not depend on terminology associated with another franchise.

---

# Mega-Morphosis Design Philosophy

## Core Transformation Goal

A successful Mega-Morphosis must satisfy two requirements simultaneously:

1. It must be immediately recognizable as the established creature.
2. It must look distinctly transformed, more powerful, and capable of abilities beyond its normal state.

Neither requirement may erase the other.

The result should communicate:

> This is unquestionably the same creature, but its defining nature has been pushed into an extraordinary temporary combat state.

## Identity Anchors

Before designing the transformed form, identify the base creature's most important identity anchors.

Possible anchors include:

* dominant silhouette rhythm
* primary body organization
* facial or sensory structure
* signature appendage
* characteristic marking
* core palette relationship
* material language
* locomotion method
* posture
* elemental behavior
* ecological concept
* personality
* recurring geometric shape

Preserve enough anchors for immediate recognition.

Do not mechanically preserve every detail. Some traits may be reshaped, intensified, divided, fused, extended, armored, exposed, or functionally repurposed when the relationship remains clear.

## Transformation Thesis

Every Mega-Morphosis must have one concise transformation thesis.

Examples:

* a cautious burrower becomes an immovable seismic bastion
* a gliding ambusher becomes a supersonic aerial interceptor
* a soft fungal colony becomes a coordinated spore-network organism
* a defensive crystal creature becomes a refractive artillery platform
* a playful electrical creature becomes a precision storm-channeling predator

The thesis should define how the creature becomes stronger.

Do not begin by asking what decorations can be added. Begin by deciding what capability, instinct, material property, or battle role has been amplified.

## Power Through Functional Change

Express increased power primarily through purposeful structural development.

Possible methods include:

* redistributing body mass
* changing the center of gravity
* refining locomotion
* expanding an existing energy-producing structure
* exposing or reinforcing internal systems
* developing a specialized offensive organ
* developing a specialized defensive structure
* converting a passive feature into an active battle feature
* changing material state
* increasing control rather than raw size
* improving aerodynamic, aquatic, subterranean, climbing, or floating efficiency
* concentrating the silhouette around a stronger battle role
* making controlled asymmetry more pronounced
* changing posture to communicate confidence, tension, speed, reach, or stability

Each major change should imply a gameplay or behavioral consequence.

## Distinctiveness Requirement

The transformed sprite must not look like the base sprite with minor additions.

At least two major visual systems should change meaningfully, such as:

* silhouette
* proportion
* posture
* body segmentation
* material distribution
* appendage function
* focal point
* locomotion presentation
* energy-flow presentation
* defensive or offensive structure

The transformation should remain recognizable even without relying on glow, aura, particle effects, or a different background.

## Recognition Requirement

The transformed form must preserve enough of the base creature's identity that a viewer can connect the two without explanatory text.

Preserve at least three meaningful identity anchors when the base design provides them.

For extremely abstract creatures, recognition may instead rely on:

* repeated spatial arrangement
* repeated material behavior
* repeated motion logic
* repeated color hierarchy
* repeated focal structure
* repeated modular relationship

Recognition does not require preserving conventional anatomy.

## Power-Specific Uniqueness

Different creatures should not all receive the same kind of Mega-Morphosis.

Do not repeatedly solve transformation by adding:

* spikes
* horns
* larger wings
* floating crystals
* glowing lines
* armor plates
* longer claws
* flame-like hair
* extra tails
* generic energy auras

Any of these may be appropriate for one creature, but none represents the global Mega-Morphosis style.

The transformation method must emerge from the individual creature's:

* anatomy
* material
* behavior
* habitat
* battle role
* temperament
* existing weaknesses
* elemental or conceptual identity

## Intensity Without Visual Noise

A Mega-Morphosis may be more elaborate than the base form, but it remains a game sprite.

Use hierarchy:

1. one dominant transformed idea
2. one supporting structural change
3. one or two restrained accent changes

Remove details that compete with the dominant idea.

Power should be communicated through stronger design decisions, not a greater number of unrelated details.

## Palette Relationship

The Mega-Morphosis should normally inherit the base creature's palette logic.

It may:

* deepen dominant colors
* shift value contrast
* introduce one controlled accent
* expose a previously hidden material color
* redistribute existing colors
* alter saturation to reflect an activated state
* change the balance between outline, shadow, base, and highlight

Do not replace the full palette without a concept-specific reason.

The transparency key remains forbidden in all foreground artwork.

## Effects Restraint

Temporary transformation energy may be implied with restrained sprite-readable effects when requested.

Effects must:

* support the silhouette rather than obscure it
* remain separable from the body
* use hard-edged pixel clusters
* remain consistent between front and rear views
* avoid the exact transparency-key color
* remain optional rather than carrying the entire transformation

Do not use effects to compensate for an underdeveloped body design.

## Base-to-Mega Continuity Audit

Before approval, compare the base and Mega-Morphosis forms directly.

Confirm:

* the same core creature identity is visible
* inherited structures transform logically
* retained markings occupy believable corresponding regions
* newly introduced structures have a clear origin or relationship
* the transformed form's scale is intentional
* the battle role appears enhanced or specialized
* the design differs beyond superficial decoration
* the result does not resemble an existing copyrighted transformation

# Mega-Morphosis Generation Protocol

When the user requests a Mega-Morphosis of an existing creature, use this workflow.

## Step M1: Lock the Base Creature

Treat all approved base-creature traits as established canon.

Identify:

* body organization
* silhouette anchors
* proportions
* materials
* palette hierarchy
* markings
* locomotion
* personality
* battle role
* elemental or conceptual identity
* front/rear structural facts

Do not reinterpret or replace these facts without an explicit request.

## Step M2: Select Identity Anchors

Choose the traits that must remain immediately recognizable.

Prefer foundational traits over tiny decorative details.

## Step M3: Define the Transformation Thesis

Write one internal sentence explaining how the temporary state amplifies the creature.

The thesis must describe a functional or thematic power change, not merely an appearance change.

## Step M4: Choose the Power Expression

Select a small number of compatible transformation mechanisms.

Examples:

* increased speed through aerodynamic restructuring
* increased defense through material densification
* increased reach through controlled body extension
* increased magical output through expanded channeling structures
* increased precision through refined sensory anatomy
* increased area control through modular or colony expansion
* increased mobility through a changed locomotion system
* increased resilience through adaptive regeneration structures

## Step M5: Establish Major Visual Changes

Plan at least two meaningful changes to silhouette, anatomy, material, posture, or functional structures.

Do not rely solely on coloration or effects.

## Step M6: Preserve Recognition

Verify that at least three identity anchors remain readable, or use the abstract-creature recognition rules when three conventional anchors do not exist.

## Step M7: Simplify at Sprite Scale

Remove transformed details that become noise at native resolution.

The Mega-Morphosis should read as more powerful at thumbnail size.

## Step M8: Construct Both Views

Create front-oriented and rear-oriented three-quarter battle sprites of the transformed state.

Reconstruct every transformed feature consistently from both viewpoints.

## Step M9: Compare Against the Base Form

Confirm that the result is neither too similar nor too detached.

Revise when:

* it resembles a minor costume change
* it becomes an unrelated monster
* inherited anatomy no longer makes sense
* recognition depends only on color
* power depends only on an aura

## Step M10: Audit Originality and Export Safety

Apply all standard originality, pixel-art, layout, and transparency-key requirements.

Base creature reference:

[INSERT OR ATTACH THE APPROVED BASE CREATURE AND ITS CANONICAL BRIEF.]

Transformation direction:

[INSERT THE DESIRED POWER, BATTLE ROLE, ELEMENTAL SPECIALIZATION, OR CREATIVE DIRECTION. IF NONE IS PROVIDED, DERIVE ONE PURPOSEFUL TRANSFORMATION THESIS FROM THE BASE CREATURE'S EXISTING DESIGN.]

This is not a permanent developmental stage, a separate species, or an unrelated redesign. It is a temporary high-output battle state of the same creature.

Preserve the base creature's most important identity anchors, including its foundational body logic, recognizable silhouette traits, core palette relationship, material language, defining markings, sensory or facial identity, and established personality where applicable.

Make the transformed form distinctly different and visibly more powerful. Introduce at least two meaningful changes to silhouette, proportions, posture, anatomy, material distribution, locomotion presentation, offensive structures, defensive structures, sensory systems, or energy-channeling structures.

Every major change must logically amplify the creature's existing concept or give it a clear specialized battle capability. Do not simply make it larger, recolor it, cover it in generic armor, add random spikes, or depend on glow effects to communicate power.

Place both sprites on a completely flat solid background using exactly `#FF00FF`, RGB `255, 0, 255`.

No part of the creature, transformed anatomy, outline, shading, highlight, marking, accessory, detached component, or power effect may contain exact RGB `255, 0, 255`.

Do not reproduce, trace, recolor, combine, or closely resemble an existing copyrighted creature or franchise-specific transformation. Do not create a recognizable character fusion.

The final design must feel like a unique combat apex of this specific creature—not a reusable generic power-up template.

# Color-Key Transparency Standard

Unless another export format is explicitly requested, use a completely flat background of:

`#FF00FF`

RGB:

`255, 0, 255`

This color represents transparent space.

## Mandatory Rules

* The complete background must be exact `#FF00FF`.
* No gradient is permitted.
* No floor shadow is permitted.
* No decoration is permitted.
* No texture is permitted.
* No border is permitted.
* No text is permitted.
* No foreground pixel may use exact RGB `255, 0, 255`.
* No semi-transparent edge pixel may blend with the background.

When pink, purple, or magenta is important to the creature, select a visibly different RGB value.

Before approval, inspect for accidental key-color pixels inside the foreground artwork.

---

# Sprite Sheet Layout

For a two-view battle sheet:

* place the front-oriented sprite on the left
* place the rear-oriented sprite on the right
* provide clean independent bounding areas
* maintain consistent visual scale
* leave enough spacing for clean extraction
* prevent silhouettes or effects from touching
* do not add labels
* do not add borders
* avoid excessive unused canvas

When exact dimensions are supplied, obey them strictly.

---

# Originality Requirements

Every monster must be independently designed.

Do not:

* reproduce an existing copyrighted creature
* trace an existing sprite
* recolor an existing monster
* combine recognizable components from existing characters
* create a recognizable fusion
* imitate an iconic pose too closely
* reproduce signature markings
* use franchise-specific symbols
* include protected names or logos
* describe the output as official franchise artwork

Genre conventions may be used in original combinations.

The complete design must possess its own:

* structure
* silhouette
* visual logic
* behavior
* motif
* personality
* material language

When a design feels too close to an existing creature, change foundational characteristics rather than applying superficial edits.

# Creature-Brief Separation Rule

Before creating anything, separate instructions into two categories.

## Style Instructions

# Required Workflow

## Step 0: Identify the Requested Mode

Determine whether the request concerns:

* a new base creature
* a permanent developmental stage
* a standard form or variant
* a Mega-Morphosis
* a revision of an approved asset
* an evaluation or quality-control pass

Use the Mega-Morphosis Generation Protocol when the requested form is a temporary empowered transformation. Do not process it as an ordinary developmental stage.

## Step 1: Read the Current Brief

Extract explicitly required traits.

Do not import anatomy from previous creatures.

## Step 2: Establish Creative Freedom

Identify which physical decisions remain open.

When most decisions are open, deliberately choose a body organization rather than defaulting to a familiar pet-like monster.

## Step 3: Define the Creature's Core

Select:

* one structural concept
* one movement concept
* one behavioral concept
* one material concept
* one visual motif

Ensure these ideas support one another.

## Step 4: Check Recent-Design Bias

Compare the concept with recently generated creatures.

Reduce repeated use of the same:

* anatomy
* locomotion
* appendages
* silhouette
* facial system
* material
* posture

## Step 5: Simplify for Sprite Scale

Remove details that do not survive at native resolution.

## Step 6: Establish Palette Ramps

Build a limited palette appropriate to this creature.

## Step 7: Construct the Front View

Create a readable three-quarter battle presentation.

## Step 8: Construct the Rear View

Reconstruct the exact same creature from behind.

## Step 9: Audit Structural Consistency

Check only the features that actually exist.

## Step 10: Audit Style

Verify pixel construction, palette, outlines, shading, and readability.

## Step 11: Audit Originality

Revise foundational traits when the design resembles an existing creature too closely.

## Step 12: Audit Export Safety

Verify the background, transparency key, spacing, scale, and absence of artifacts.

# Reusable Generation Prompt Template

Create an entirely original, game-ready monster sprite sheet in a polished late-2000s handheld monster-catching RPG pixel-art aesthetic.

Creature brief:

[INSERT THE CURRENT CREATURE BRIEF.]

Where the brief leaves physical design open, take meaningful creative liberty. Independently decide the creature's body organization, number of limbs or lack of limbs, locomotion, sensory system, material, silhouette, posture, and biological or non-biological structures.

Do not assume the creature should be mammalian, humanoid, bipedal, quadrupedal, dragon-like, pet-like, symmetrical, or conventionally cute.

Do not automatically add wings, horns, tails, ears, paws, claws, crests, spikes, shells, cheek markings, elemental symbols, or glowing gems. Include any such feature only when it directly strengthens the current creature concept.

Favor a coherent and surprising body plan over a familiar animal body decorated with extra features.

Create:

* one front-oriented three-quarter battle sprite
* one rear-oriented three-quarter battle sprite

Both views must depict the same creature with identical structure, proportions, materials, patterns, palette, lighting direction, outline treatment, and visual scale.

Reconstruct the creature accurately from both angles. Do not invent or remove structures between views. If the creature lacks conventional front-and-back anatomy, interpret the two views as logically opposing battle viewpoints.

Use:

* a bold readable silhouette
* proportions appropriate to the specific creature
* crisp one-pixel outlines at native sprite scale
* deliberate hand-placed pixel clusters
* clean stair-stepped curves
* a limited vibrant palette
* two-to-four-step color ramps
* compact cel-style shading
* restrained texture
* only details that remain readable at actual sprite size
* authentic hand-pixelled construction

The artwork must look designed directly as pixel art. It must not look like smooth artwork that was downscaled or passed through an automatic pixel filter.

Place both sprites with clean separation on a completely flat, solid background using exactly `#FF00FF`, RGB `255, 0, 255`.

No part of the monster, outline, shading, highlight, sensory feature, pattern, accessory, effect, or other foreground element may contain exact RGB `255, 0, 255`.

Do not use anti-aliasing, semi-transparent pixels, gradients, smooth vector lines, painterly rendering, realistic microtexture, 3D effects, blurry pixels, modern high-resolution anime rendering, automatic pixel-filter artifacts, excessive dithering, arbitrary elemental motifs, unnecessary decorative structures, text, logos, signatures, watermarks, cast shadows, floor shadows, or background decoration.

Do not reproduce, trace, recolor, combine, or closely resemble an existing copyrighted creature. Do not create a recognizable character fusion.

Keep the monster cohesive, original, structurally believable within its own rules, visually distinct from previously generated creatures, and suitable for direct use in the same creature-battling game.

# Revision Protocol

When revising an existing monster, preserve every approved design choice except the traits explicitly targeted by the revision.

Do not use the creative-liberty rules as permission to redesign an already approved monster.

## Preserve

Identify all approved:

* structural traits
* proportions
* materials
* palette decisions
* pose elements
* patterns
* focal features
* sprite scale
* style decisions

## Change

Apply only the requested modifications.

## Verify

Check that the edit did not unintentionally alter unrelated features or introduce conventional anatomy that the creature did not previously possess.

## Mega-Morphosis Revisions

When revising a Mega-Morphosis, preserve both:

* the approved identity relationship to the base creature
* the approved transformation thesis and power specialization

Do not weaken recognition while making the form more dramatic. Do not reduce distinctiveness while restoring inherited traits. Change only the requested transformed features unless the current design fails a mandatory continuity, originality, readability, or export rule.

# Quality-Control Checklist

## Style

* [ ] The artwork reads as native pixel art.
* [ ] Pixel clusters appear intentionally placed.
* [ ] The silhouette is readable.
* [ ] The palette is limited and cohesive.
* [ ] Shading uses compact clusters.
* [ ] Outlines are crisp and controlled.
* [ ] No smooth or painterly rendering remains.
* [ ] Important features survive at native scale.

## Creative Design

* [ ] The body plan was selected intentionally.
* [ ] No conventional anatomy was added by default.
* [ ] Optional appendages exist only when they strengthen the concept.
* [ ] The creature differs structurally from recent designs.
* [ ] The design has one clear primary idea.
* [ ] Secondary features support rather than compete with that idea.
* [ ] The monster does not feel like a familiar animal with random decorations.
* [ ] The design is surprising without becoming incoherent.

## Structure

* [ ] All existing structures relate coherently.
* [ ] The locomotion method fits the anatomy.
* [ ] The center of mass fits the pose.
* [ ] Floating or disconnected structures are visibly intentional.
* [ ] No body region appears accidentally detached.
* [ ] The design can be mentally rotated between views.

## View Consistency

* [ ] Both views depict the same creature.
* [ ] Proportions match.
* [ ] Existing patterns match.
* [ ] Materials match.
* [ ] Structural counts match.
* [ ] Placement matches.
* [ ] Lighting matches.
* [ ] Outline treatment matches.
* [ ] Sprite scale matches.
* [ ] The rear view does not invent conventional anatomy.

## Mega-Morphosis

* [ ] The form is explicitly a temporary empowered state.
* [ ] The base creature remains immediately recognizable.
* [ ] At least three identity anchors remain readable, or the abstract-recognition rule is satisfied.
* [ ] At least two major visual systems changed meaningfully.
* [ ] The transformation has one clear functional thesis.
* [ ] Every major new feature supports the amplified battle role.
* [ ] Power is communicated without depending entirely on effects.
* [ ] The form is not merely larger, recolored, armored, or covered in spikes.
* [ ] The transformed silhouette is distinct at thumbnail size.
* [ ] The palette remains related to the base form unless a justified concept requires otherwise.
* [ ] Front and rear views agree on all transformed structures and effects.
* [ ] The design does not reuse a generic Mega-Morphosis template from another creature.

## Technical

* [ ] Background is exactly `#FF00FF`.
* [ ] Foreground contains no exact `#FF00FF`.
* [ ] No anti-aliased pixels are present.
* [ ] No semi-transparent pixels are present.
* [ ] No cast shadow appears.
* [ ] Sprites have clean separation.
* [ ] No text, logo, signature, or watermark appears.
* [ ] Dimensions follow the current asset brief.
* [ ] Sprites can be isolated cleanly.

# Failure Conditions

Reject or revise the output when:

* the creature resembles an existing copyrighted monster
* the design defaults to familiar anatomy without justification
* wings, horns, tails, ears, or similar structures were added merely from habit
* the creature repeats the dominant body plan of several recent monsters
* the design is a conventional animal with unrelated decorations
* the artwork looks vector-drawn or automatically pixelated
* front and rear views contradict one another
* the pose conceals the core structure
* the background is not exact `#FF00FF`
* the foreground contains exact `#FF00FF`
* the sprite uses anti-aliasing or semi-transparency
* important details vanish at native size
* lighting or outline treatment changes between views
* the output contains text, backgrounds, shadows, or watermarks
* a Mega-Morphosis is treated as a permanent developmental stage
* a Mega-Morphosis is only a recolor, size increase, aura, or minor accessory change
* a Mega-Morphosis becomes unrecognizable as the base creature
* a Mega-Morphosis remains too visually similar to the base form
* transformed features have no functional relationship to the creature's identity
* the same generic spikes, armor, wings, crystals, or energy lines are reused across unrelated Mega-Morphoses
* the transformation copies recognizable franchise-specific design language

# Agent Behavior

When this skill is active, the agent must:

1. Treat the current creature brief as the authority for required species traits.
2. Treat this skill as the authority for style and production quality.
3. Exercise real creative liberty when anatomy is unspecified.
4. Avoid defaulting to pet-like, mammalian, dragon-like, winged, horned, or tailed forms.
5. Consider zero limbs, unusual limb counts, unconventional sensory systems, and non-animal materials as normal options.
6. Preserve approved creature traits during revisions.
7. Favor a clear structural concept over feature accumulation.
8. Compare new random designs with recent creatures to reduce repetition.
9. Use absence as a valid design choice.
10. Keep every creature readable at native sprite scale.
11. Distinguish art-style consistency from anatomical repetition.
12. Never claim exact pixel compliance without inspecting the resulting asset.
13. Flag technical limitations when an image model cannot guarantee exact RGB values or true native pixel placement.
14. Treat Mega-Morphosis as a temporary empowered battle state, never as an ordinary developmental stage.
15. Preserve recognizable identity anchors while making every Mega-Morphosis substantially distinct.
16. Give each Mega-Morphosis a creature-specific transformation thesis and battle specialization.
17. Never use one generic power-up anatomy across the roster.
18. Reject transformations whose power is communicated only through size, recoloring, spikes, armor, or effects.

# Final Standard

Every monster should feel unique in structure, behavior, and visual identity while unmistakably belonging to the same game.

Consistency must come from:

* shared pixel craftsmanship
* shared palette discipline
* shared shading logic
* shared outline behavior
* shared battle staging
* shared technical standards
* shared visual clarity

Consistency must never depend on repeatedly using:

* the same body plan
* the same limb count
* the same face
* the same proportions
* the same appendages
* the same materials
* the same pose
* the same motifs
* the same palette

The roster should look like a diverse living universe—not variations of one creature template.

Its Mega-Morphoses should look like distinct combat apices of individual creatures—not variations of one universal transformation template.

Every monster may obey different anatomical rules.

The art direction must remain unified.
